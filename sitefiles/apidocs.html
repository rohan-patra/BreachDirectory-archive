API Documentation


This can be used for anything from looking up breached emails and their sources to getting a list of data acquired on a specific domain breach.

Authorization

The first thing that you will need is an API key which you can get by contacting us at rohanpatra2003@gmail.com.

This key will need to be embedded in your API request via the "key" GET request variable.

For example, the beginning of your request should if your api key was: jYKhkmBTP89CUMetYmdnVz9chztyTVYy
you're request would be:

curl -X GET "https://breachdirectory.tk/api?key=jYKhkmBTP89CUMetYmdnVz9chztyTVYy"

Usage
Our API supports searching the database using 6 separate functions
  • "email" - lookup passwords, hashes, and sources associated with a given email
  • "user" - lookup passwords, hashes, and sources associated with a given username
  • "pass" - find out how many times a specific password was exposed / we are looking to implement their sources in the future as well
  • "domain" - lookup emails exposed from a specific domain (requires ownership verification)
  • "hash" - same as the email function but accepts the MD5, SHA-1, or SHA-256 hashed version of the email to protect the anonymity of the request
  • "auto" - automatically detect either an email or username search term and search its respective database
  • "pastes" - search pastes for a given email (results are returned in the same format as the email function)

The function type is accepted in the "func" GET request variable, and the actual lookup is accepted in the "term" variable.

A sample request for the email "test@example.com" using the above api key is shown below.

curl -X GET "https://breachdirectory.tk/api?key=jYKhkmBTP89CUMetYmdnVz9chztyTVYy&func=email&term=test@example.com
